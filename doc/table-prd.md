# Table 增删改查 PRD

## 1. 文档信息
- 文档名称：Table 增删改查需求文档
- 版本：V1.0
- 日期：2026-06-20

## 2. 背景
在日常业务管理中，用户需要通过一个简单清晰的表格页面，对数据进行查看、新增、修改和删除操作，以提升信息维护效率，减少重复沟通和人工处理成本。

## 3. 目标
- 提供一个基础 Table 页面，支持数据查询与管理
- 支持新增、编辑、删除、查看完整数据列表
- 新增和编辑操作使用弹窗完成，降低页面跳转成本
- 保持交互简单，便于快速上线

## 4. 叙事
运营人员进入数据管理页面后，可以直接看到当前所有记录。若需要新增一条数据，点击“新增”按钮后，页面弹出表单弹窗，填写必要信息并提交即可完成创建。若发现某条数据需要调整，点击该行“编辑”按钮后，弹窗展示原有内容，用户修改后保存即可更新。若某条数据不再需要，点击“删除”按钮后进行二次确认，确认后系统删除该条记录。整个过程尽量在当前页面完成，避免频繁跳转，提升处理效率。

## 5. 功能范围

### 5.1 列表查询
- 页面默认展示数据列表
- 以 Table 形式展示记录
- 每行至少包含基础字段与操作列
- 支持空状态展示

### 5.2 新增
- 页面提供“新增”按钮
- 点击后打开新增弹窗
- 弹窗内包含表单字段
- 用户点击“确认”后进行保存
- 保存成功后关闭弹窗并刷新列表

### 5.3 编辑
- 每行提供“编辑”按钮
- 点击后打开编辑弹窗
- 弹窗内默认回填当前行数据
- 用户修改后点击“确认”完成更新
- 更新成功后关闭弹窗并刷新列表

### 5.4 删除
- 每行提供“删除”按钮
- 点击后弹出二次确认提示
- 用户确认后删除该条记录
- 删除成功后刷新列表

## 6. 页面与交互要求
- 页面结构：标题区 + 操作区 + Table 列表区
- 操作区至少包含“新增”按钮
- Table 操作列至少包含“编辑”和“删除”
- 新增、编辑统一使用弹窗
- 弹窗按钮包含“取消”和“确认”
- 删除必须有确认提示，避免误删
- 成功操作后显示成功提示
- 失败操作后显示失败提示及原因

## 7. 字段示例
以下字段仅作为默认示例，可按实际业务替换：

| 字段 | 说明 | 是否必填 |
| --- | --- | --- |
| name | 名称 | 是 |
| type | 类型 | 是 |
| status | 状态 | 否 |
| remark | 备注 | 否 |

## 8. 验收标准
- 用户可以看到数据列表
- 用户可以通过弹窗新增一条记录
- 用户可以通过弹窗编辑一条已有记录
- 用户可以删除一条记录，且删除前有确认提示
- 新增、编辑、删除完成后，列表结果正确更新
- 页面在无数据时有明确空状态提示

## 9. 非功能要求
- 页面风格简洁清晰
- 常规操作响应流畅
- 弹窗表单校验明确，避免错误提交
- 交互文案统一，便于理解

## 10. 版本边界
本期仅包含基础增删改查能力，不包含：
- 批量操作
- 导入导出
- 复杂筛选
- 权限细分
- 操作日志
